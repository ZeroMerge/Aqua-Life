# AquaLife — Aquarium Monitoring Dashboard

A professional, enterprise-grade aquarium monitoring dashboard for real-time water quality management. Monitors four critical sensors — pH, Water Temperature, Turbidity, and Dissolved Oxygen — with live video streaming, real-time analytics, and threshold reference data.

## Production Deployment Guide

### Prerequisites

- Node.js 20+
- A Supabase project (optional — the app falls back to realistic demo data if no Supabase config is provided)
- A Cloudflare Tunnel or other streaming endpoint for the MJPEG video feed (optional)

### Environment Variables

Create a `.env` file in the project root:

```bash
# Supabase Configuration (required for production data)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key

# MJPEG Video Stream URL (optional)
VITE_STREAM_URL=https://your-tunnel.trycloudflare.com/stream
```

| Variable | Description | Required |
|---|---|---|
| `VITE_SUPABASE_URL` | Your Supabase project URL | No (falls back to demo data) |
| `VITE_SUPABASE_ANON_KEY` | Your Supabase anon/public key | No (falls back to demo data) |
| `VITE_STREAM_URL` | MJPEG stream endpoint URL | No (shows offline state if absent) |

### Supabase Setup

1. Create a new project at [supabase.com](https://supabase.com)
2. In the SQL Editor, run:

```sql
create table sensor_logs (
  id bigint generated by default as identity primary key,
  ph double precision not null,
  temperature double precision not null,
  turbidity double precision not null,
  dissolved_oxygen double precision not null,
  created_at timestamptz default now() not null
);

-- Enable realtime
alter publication supabase_realtime add table sensor_logs;
```

3. Go to Project Settings > API to get your `URL` and `anon` key.

### Sensor Data Ingestion

The app subscribes to Supabase realtime INSERT events. To feed data into the system, insert rows into `sensor_logs`:

```sql
insert into sensor_logs (ph, temperature, turbidity, dissolved_oxygen)
values (7.2, 26.5, 25, 8.5);
```

In production, this would typically come from an IoT gateway or edge device with water quality probes.


### Production Build

```bash
npm run build
```

Output is in `dist/` — static files ready for any CDN or static host.

### Deployment Options

**Static Hosting (Recommended)**

Deploy `dist/` to any static host:
- Vercel: `vercel --prod`
- Netlify: Drag `dist/` folder to deploy
- Cloudflare Pages: Connect Git repo
- AWS S3 + CloudFront: Upload `dist/` contents
- GitHub Pages: Use `gh-pages` branch

**Important**: Environment variables must be set at **build time** since they are embedded by Vite. For hosts that support build-time env vars, configure them in the host's dashboard.

### Architecture

- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS
- **Charts**: Recharts (real-time line charts)
- **Data**: Supabase JS client with realtime subscriptions
- **Video**: MJPEG stream via `<img>` tag with reconnection logic
- **State**: React hooks with rolling average computation

### Sensor Thresholds (Catfish)

| Sensor | Safe Range | Warning Zone | Critical |
|---|---|---|---|
| pH | 6.5 – 8.0 | < 6.65 or > 7.85 | < 6.0 or > 8.5 |
| Temperature | 24 – 30°C | < 21.4 or > 28.6 | < 20 or > 34 |
| Turbidity | 0 – 50 NTU | > 47 | > 80 |
| Dissolved Oxygen | 5 – 12 mg/L | < 5.2 | < 3 |

Warning triggers when a value is within 10% of any safe boundary. Status is computed on a rolling average of the last 5 readings to prevent flickering from momentary spikes.

### Design System

The UI follows IBM Carbon Design System and GitHub Primer conventions:
- Strictly monochromatic palette (white, grays, black)
- Single accent blue (`#0969da`) for active states and primary actions
- Three status colors only on indicators (green/amber/red)
- No gradients, no decorative shadows, no marketing copy
- Monospace font for sensor numeric values
